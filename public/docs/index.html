Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Endpoints

POST api/register

Example request:
curl --request POST \
    "http://localhost/api/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"name\": \"e\",
    \"email\": \"vbrown@example.com\",
    \"password\": \"I\\\"dkrB[(nw%K69=Nx#5H\"
}"

const url = new URL(
    "http://localhost/api/register"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "name": "e",
    "email": "vbrown@example.com",
    "password": "I\"dkrB[(nw%K69=Nx#5H"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

POST api/register

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

name   string   

Must not be greater than 255 characters. Example: e

email   string   

Must be a valid email address. Must not be greater than 255 characters. Example: vbrown@example.com

password   string   

Must be at least 8 characters. Example: I"dkrB[(nw%K69=Nx#5H

Login user and return a token.

Example request:

    "http://localhost/api/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \

    \"email\": \"fisher.justen@example.org\",
    \"password\": \"7|JSZ$^E3g9xKU42P\"
}"
